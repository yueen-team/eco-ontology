# API And Contract Interfaces

This repository has no runtime HTTP API yet.

The API surface is contract-oriented:

- JSON Schemas under `schemas/`.
- Versioned bundles under `contracts/`.
- Registries under `registries/`.
- Generated projections under `dist/projections/`.
- Release manifests that downstream systems can pin by version and sha256.

## First Contract Interfaces

- `semantic_event.v2`: EcoCheck outgoing field fact candidate payload.
- `graph_node`: eco-execution-graph node export contract.
- `graph_edge`: eco-execution-graph edge export contract.
- `graph_source`: eco-execution-graph source export contract.
- `kb_product_manifest`: eco-semantic-knowledge-base approved package manifest.
- `ontology_registry.v1`: common registry entry shape.
- `release_manifest.v1`: v0.1.0 artifact/hash manifest.
- `consumer_compatibility_matrix.v1`: three-consumer adoption matrix.
- `projections.*.v1`: generated projection artifact shapes.

## Compatibility Rule

Any breaking contract change requires an ADR and a new major ontology package
version.

## Projection Interfaces

- `dist/projections/ecocheck/ontology-contracts.generated.json`
- `dist/projections/graph/ontology-registry.generated.json`
- `dist/projections/graph/schema.fragment.generated.json`
- `dist/projections/kb/ontology-registry.generated.json`
- `dist/projections/kb/schema.fragment.generated.json`

Projection files are generated by `pnpm projections:generate`; consumers should
pin them by the sha256 values in `contracts/release-manifest.v1.json` and
`contracts/consumer-compatibility-matrix.v1.json`.

The generated projection artifact interface is validated by the
`schemas/projections.*.v1.schema.json` family. Hash checks prove deterministic
content; the schemas prove the shape that consumers can read.

### Projection Schemas

The projection governance middle layer validates generated artifacts with
Draft-07 / Ajv v6 compatible schemas:

- `schemas/projections.ecocheck.v1.schema.json` for the EcoCheck contract/id
  projection.
- `schemas/projections.registry.v1.schema.json` for graph and KB registry
  projections.
- `schemas/projections.schema_fragment.v1.schema.json` for graph and KB schema
  fragments.
- `schemas/projections.manifest.v1.schema.json` for
  `dist/projections/projection-manifest.v1.json`.

Checks:

- Every projection schema declares Draft-07 and compiles with Ajv v6.
- Each generated artifact validates against its matching projection schema.
- `generated_by.source_sha256` records all generator source schema and registry
  inputs with lowercase 64-character sha256 values.
- Registry and enum lists are non-empty and unique.
- Ownership boundaries stay consumer-specific: graph keeps assembly/tiering/
  review/export fields, KB keeps approved atom/source manifest/knowledge product
  fields, and ontology keeps shared ids, schema refs, manifest schema, release
  hashes, and projection hashes.
- The projection manifest lists exactly the five generated consumer projection
  artifacts and records a sha256 for each.
