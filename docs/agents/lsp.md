# LSP Guidance

## Runtime

- Node.js is used for local verification scripts and Git workflow hooks.
- JSON Schema files should stay under `schemas/`.
- Markdown documentation is source material and should be indexed by editor
  search.

## Editor Expectations

- Enable JavaScript/Node language service for `scripts/**/*.mjs` and
  `verify/**/*.mjs`.
- Enable JSON language support for `schemas/**/*.json`,
  `contracts/**/*.json`, `registries/**/*.json`, and
  `verify/afk-test.config.json`.
- Enable Markdown validation and link checking where available.

## Generated Or Local Artifacts

- `reports/` is for local validation evidence and is ignored except
  `reports/.gitkeep`.
- `bdd/behavior-contracts.ndjson` is generated by `pnpm bdd:export`.
